ARIA Examples

ARIA has a plethora of ways it can be implemented to enhance web accessibility via attributes. The main 3 attributes are roles, states, and properties.

Roles are used when defining the type of interactive element. Example being buttons, checkbox, and slider.

States are used to indicate the current condition/state of an element. Most commonly used by buttons. Below is a state example of a button.

<button aria-pressed="false" aria-label="Play video" onclick="togglePlay(this)">Play</button>

Finally, properties are used to define additional information about an element that is not covered by roles or states. They are typically seen as static and mostly used with labels.
